# Performance v8.6.0

## Método

Tempos são medidos na mesma máquina Windows usada no baseline v0.8.5, com `Stopwatch`. Tamanhos
vêm de `frontend/dist/assets`. Não há SLA nem extrapolação para produção.

| Medida | v0.8.5 | v8.6.0 | Observação |
| --- | ---: | ---: | --- |
| Pytest | 93 testes, 20,63 s internos | 94 testes, 23,50 s internos/26,98 s do gate | variação local; sem warnings |
| Frontend unit + build | 8,90 s (build isolado) | 15,82 s com lint + Vitest + build; Vite 4,13 s | escopo agora inclui testes |
| Playwright | inexistente | 10 testes em 18,4 s; gate 19,33 s | Chromium, serviços reais locais |
| Maior chunk | 329.967 B | 329.967 B antes do redesign adicional | lazy routes preservadas |
| Auditoria | página SQL sem total | página + count coerente | duas queries limitadas |

Tempos de endpoint e número total de queries não são publicados sem instrumentação reproduzível.
Na listagem de pacientes, identificadores passaram de uma consulta por paciente para uma consulta
em lote. A auditoria executa uma query de count e uma query limitada por página.
