# Baseline de desempenho v0.8.5

Medição local em Windows, Python 3.12 e Node 24, antes das alterações, em 11/07/2026. Valores não
representam produção e não devem ser usados como SLA.

| Medida | Antes (v0.8.4) | Depois (v0.8.5) | Melhoria | Pendência |
| --- | ---: | ---: | --- | --- |
| Pytest completo | 21,19 s (88 testes) | 20,63 s internos; 23,00 s de parede (93 testes) | +5 testes, tempo interno -0,56 s | ambiente local variável |
| Comando de build | 11,36 s | 8,90 s | -2,46 s nesta execução | inclui TypeScript + Vite |
| Maior chunk JS | 328.157 B | 329.967 B | +1.810 B (+0,55%) | custo pequeno do polish; vendor ainda compartilhado |
| Chunk `schemas` | 96.676 B | 96.676 B | estável | avaliar importação granular na v0.9.0 |
| Auditoria para página N | carregava todos os eventos e filtrava em memória | SQL com `WHERE/OFFSET/LIMIT` | evita O(n) na aplicação | JSON textual no SQLite é limitação |

## Método

Tempos foram medidos com `Stopwatch` ao redor dos comandos oficiais. Tamanhos vieram de
`frontend/dist/assets`. A auditoria SQL foi confirmada por inspeção e teste com filtro + segunda
página. Não foram inventados tempos de endpoint ou quantidade de queries sem instrumentação
reprodutível.
