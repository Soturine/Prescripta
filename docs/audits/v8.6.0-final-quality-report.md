# Relatório final de qualidade v8.6.0

- qualidade textual, links e assets: aprovados;
- `pip check` e Ruff: aprovados;
- backend: 94 testes, zero warnings;
- frontend: dois testes Vitest, typecheck e build aprovados;

## Correções objetivas

O teste que chamava `powershell` usa `sys.executable`; CI executa backend em Ubuntu e Windows. O
release-preflight espera o workflow do SHA enviado. HTTP 422 foi atualizado, o conjunto Starlette
recebeu `httpx2`, e um erro timezone-aware/naive descoberto pelo E2E foi corrigido.

Auditoria agora retorna total e metadados; paciente carrega identificadores em lote. README e docs
foram conectados a evidências v8.6.0 reais.

- unidade de trabalho com commit único em todos os casos de uso;
- eliminação de todo N+1 do catálogo e relatórios;
- testes unitários de todos os componentes visuais;

- tokens para providers que não retornam usage e custo monetário configurável;
