# Prescripta v0.8.2

Release de protocolos rápidos, README de produto, frontend polish e assets
públicos mais completos.

## Auditoria Inicial

- README estava curto e com cara de resumo de release.
- Benchmarks ainda apontavam protocolos de emergência como lacuna.
- Assets cobriam poucas telas.
- Frontend precisava de hierarquia visual, estados de loading e área de decisão
  rápida.
- Docs internas de onboarding, UX e protocolos precisavam ficar mais ricas.

## Destaques

- Nova aba **Protocolos** com sete fluxos rápidos demonstrativos.
- Execução de protocolo com contexto mínimo, flags, cálculos demonstrativos e
  auditoria `protocol.run`.
- Explicação de protocolo por IA/fallback sem alterar passos, dose, fonte ou
  decisão.
- Relatório, PDF, JSON e CSV para evento de protocolo.
- README raiz reestruturado como material de produto e instalação.
- Polish visual no frontend: botões, campos, loading, empty state, sidebar,
  dashboard e nova tela de protocolos.
- Docs novas de protocolos, política de fontes, UX, quickstart, troubleshooting e
  jornada inicial.
- Benchmark v0.8.2 registra fechamento da lacuna de protocolos.

## Segurança

- Protocolos são educacionais e exigem julgamento humano.
- IA não decide conduta e não inventa evidência.
- Eventos de protocolo registram `secret_logged=false`.
- Não foram usados dados reais de paciente.

## Testes

- Cobertura para list/detail/run de protocolos.
- Validação de contexto mínimo.
- Auditoria e exportação do evento.
- Report preview e explicação com fallback.

## Próximos Passos

`v0.9.0` deve priorizar Docker, PostgreSQL, migrações e deploy demo. Uma
`v0.8.3` só faz sentido se houver ajustes clínicos/UX finos antes da migração.
