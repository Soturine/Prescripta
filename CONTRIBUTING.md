# Contributing

Obrigado por contribuir com Prescripta.

## Fluxo

1. Crie uma branch curta e descritiva.
2. Faça commits usando Conventional Commits.
3. Rode testes e lint antes de abrir PR.
4. Atualize documentação quando alterar comportamento.
5. Atualize `CHANGELOG.md` em mudanças relevantes.

## Qualidade

- Backend deve manter regras em `services`.
- Frontend não deve conter regra clínica.
- Dados de exemplo devem ser fictícios.
- A aplicação é educacional e não deve prometer uso clínico real.

## Pull Requests

Inclua escopo, testes executados, riscos conhecidos e screenshots quando houver mudança visual.
